32 /**
33  * \class PerfVisitor
34  * \brief %Visitor for measuring performance related information
35  *
36  * This visitor used to visit the ast and associated symbol tables
37  * to measure the performance of every block in nmodl file. For
38  * every symbol in associated symbol table, read/write count
39  * is updated which will be used during code generation (to select
40  * memory types). Certain statements like useion, valence etc.
41  * are not executed in the translated C code and hence need to
42  * be skipped (i.e. without visiting children). Note that this
43  * pass must be run after symbol table generation pass.
44  *
45  * \todo
46  * - To measure the performance of statements like if, elseif
47  * and else, we have to find maximum performance from if,elseif,else
48  * and then use it to calculate total performance. In the current
49  * implementation we are doing sum of all blocks. We need to override
50  * IfStatement (which has all sub-blocks) and get maximum performance
51  * of all statements recursively.
52  *
53  * - In order to avoid empty implementations and checking
54  * start_measurement, there should be "empty" ast visitor from
55  * which PerfVisitor should be inherited.
56  */
