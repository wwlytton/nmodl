57 /**
58  * \class DUInstance
59  * \brief Represent use of a variable in the statement
60  *
61  * For a given variable, say tau, when we have statement like `a = tau + c + tau`
62  * then def-use is simply `[DUState::U, DUState::U]`. But if we have if-else block
63  * like:
64  *
65  * \code{.mod}
66  * IF (...) {
67  * a = tau
68  * tau = c + d
69  * } ELSE {
70  * tau = b
71  * }
72  * \endcode
73  *
74  * Then to know the effective result, we have to analyze def-use in `IF` and `ELSE`
75  * blocks i.e. if variable is used in any of the if-elseif-else block then it needs
76  * to mark as `DUState::U`. Hence we keep the track of all children in case of
77  * statements like if-else.
78  */
79 class DUInstance {
80  public:
81  /// state of the usage
82  DUState state;
83 
84  /// usage of variable in case of if like statements
85  std::vector<DUInstance> children;
86 
87  DUInstance(DUState state)
88  : state(state) {}
89 
90  /// analyze all children and return "effective" usage
91  DUState eval();
92 
93  /// if, elseif and else evaluation
94  DUState sub_block_eval();
95 
96  /// evaluate global usage i.e. with [D,U] states of children

132 /**
133  * \class DefUseAnalyzeVisitor
134  * \brief %Visitor to return Def-Use chain for a given variable in the block/node
135  *
136  * Motivation: For global to local variable transformation aka localizer
137  * pass, we need to compute Def-Use chains for all global variables. For
138  * example, if we have variable usage like:
139  *
140  * \code{.mod}
141  * NEURON {
142  * RANGE tau, beta
143  * }
144  *
145  * DERIVATIVE states() {
146  * ...
147  * x = alpha
148  * beta = x + y
149  * ....
150  * z = beta
151  * }
152  *
153  * INITIAL {
154  * ...
155  * beta = x + y
156  * z = beta * 0.1
157  * alpha = x * y
158  * }
159  * \endcode
160  *
161  * In the above example if we look at variable beta then it's defined before it's
162  * usage and hence it can be safely made local. But variable alpha is used in first
163  * block and defined in second block. Hence it can't be made local. A variable can
164  * be made local if it is defined before it's usage (in all global blocks). We exclude
165  * procedures/functions because we expect inlining to be done prior to this pass.
166  *
167  * The analysis of if-elseif-else needs special attention because the def-use chain
168  * needs re-cursive evaluation in order to find end-result. For example:
169  *
170  * \code{.mod}
171  * IF(...) {
172  * beta = y
173  * } ELSE {
174  * IF(...) {
175  * beta = x
176  * } ELSE {
177  * x = beta
178  * }
179  * }
180  * \endcode
181  *
182  * For if-else statements, in the above example, if the variable is used
183  * in any of the if-elseif-else part then it is considered as "used". And
184  * this is done recursively from innermost level to the top.
185  */
